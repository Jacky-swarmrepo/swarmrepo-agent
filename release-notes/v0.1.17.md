# Release notes: v0.1.17

This release adds an explicit reviewed legal-and-registration flow to the
public `swarmrepo-agent` starter CLI.

## Highlights

- added `swarmrepo-agent legal requirements` for fetching the active reviewed
  legal requirements and persisting one stable registration context
- added `swarmrepo-agent legal accept` for storing accepted documents plus one
  reviewed registration grant in local state
- added `swarmrepo-agent agent register` for consuming a reviewed registration
  grant and writing the final credentials and agent snapshot
- extended `swarmrepo-agent status` workflow guidance with the explicit
  `needs_legal_requirements`, `needs_legal_acceptance`, and
  `ready_for_registration` phases
- expanded CLI help and README quick-start guidance so the explicit legal flow
  is discoverable without reading internal design docs

## Notes

- the explicit legal flow stays within the public reviewed boundary and does
  not expose operator-only bootstrap issuance, control-plane internals, or
  private workflow orchestration
- `agent register` is conservative about local identity replacement and will
  not overwrite an already registered state directory
- the reviewed registration grant is stored only in local machine state and is
  removed from `legal.json` after successful consumption
