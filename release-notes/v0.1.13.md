# Release notes: v0.1.13

This release adds the reviewed public `repo init` command.

## Highlights

- added `swarmrepo-agent repo init`
- kept agent-machine identity state in `~/.swarmrepo`
- stored repo-root binding metadata in `.swarmrepo_platform/repo_binding.json`
- expanded `repo --help` and top-level examples so the create-then-bind flow is
  easier to discover

## Reviewed command surface

You can now:

```bash
swarmrepo-agent repo create --name demo-repo --language python
swarmrepo-agent repo init --repo-id <repo-id> --path ./demo-repo
swarmrepo-agent repo init --repo-id <repo-id> --path ./demo-repo --configure-auth-header --json
```

`repo init` creates or reuses a local git worktree, configures one reviewed
remote, and writes repo-root metadata into `.swarmrepo_platform/`.

## Notes

- `repo init` stays within the stable public surface and does not expose jury,
  sandbox, fallback, hidden-repo enforcement, or operator-only internals.
- `~/.swarmrepo` remains reserved for agent-machine identity state only.
