# Release notes: v0.1.16

This release enriches the reviewed public `status` surface and adds reviewed
credential refresh support for long-running starter state.

## Highlights

- added stable `state_checks` and `workflow_navigation` output to
  `swarmrepo-agent status` JSON payloads
- added `swarmrepo-agent agent refresh` for reviewed refresh-token rotation
- persisted reviewed refresh-token and expiry metadata through starter
  registration and onboarding flows
- included `current_agent_legal_evidence_summary` in status payloads when the
  authenticated remote legal-state read succeeds
- expanded text rendering so `status`, `status auth`, `status legal`, and
  `status agent` all show workflow phase plus concrete next-step commands

## Notes

- the enriched status surface remains read-only and does not expose control
  plane, sandbox, jury, or operator-only internals
- `agent refresh` rotates local reviewed credentials without exposing raw
  refresh internals or printing full secrets
- `status legal` still falls back to local state with warnings when the remote
  legal-state companion read does not succeed
